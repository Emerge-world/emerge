How Emerge Measures Population Behavior

This page explains how the simulation turns raw run artifacts into population metrics, time-based trends, and the EBS score. It is meant to read like documentation, but it can also point at real metrics files from a run.

Weights & Biases is an optional observer for live telemetry, not the canonical source of truth for run scoring.

From Events To Metrics

Every run writes an event stream to data/runs/<run_id>/events.jsonl. The builders then read that artifact after the run completes and derive summary metrics, per-tick metrics, and EBS component scores. 

events.jsonl -> summary.json + timeseries.jsonl -> ebs.json

Population Metrics

Survival

These metrics answer how many agents started, how many survived, and how much loss the population absorbed.

Survival rate = final survivors / initial population

Action Quality

These metrics ask how often actions were resolved cleanly by the Oracle and how often parsing failed before the world could react.

Oracle success rate = successful oracle resolutions / total oracle resolutions

Innovation

These metrics show whether agents tried to invent new actions, how often those attempts were approved, and whether approved actions were later used.

Innovation approval rate = approved innovations / innovation attempts

Innovation realization rate = approved innovations later used / approved innovations

Whole Run Vs Per-Tick Reading

Whole-run metrics tell you how a run ended. Per-tick metrics tell you how the population changed over time.

The time series focuses on alive count, mean hunger, mean energy, mean life, actions, deaths, and innovation activity for each tick.

EBS Breakdown

EBS is a weighted interpretation layer built on top of the recorded artifacts, not a magic intelligence score. 

EBS =
0.30 * Novelty
+ 0.20 * Utility
+ 0.20 * Realization
+ 0.15 * Stability
+ 0.15 * Autonomy

Novelty

Measures whether approved innovations are varied and structurally different.

Utility

Measures whether approved innovations appear to create useful state changes.

Realization

Measures whether approved innovations are actually used and succeed.

Stability

Measures how coherent behavior remains across invalid actions and contradictions.

Autonomy

Measures proactive and environment-contingent behavior in the current implementation.

Limits And Interpretation

Direct counts are more concrete than heuristic scores. EBS is a structured interpretation layer based on implemented signals, not an absolute truth.

Some sub-signals are intentionally partial. For example, self_generated_subgoals is currently fixed at 0.0.

Weights & Biases remains an optional observer rather than the official record for metrics or scoring.
