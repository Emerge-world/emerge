How Emerge Measures Population Behavior

This page explains how the simulation turns raw run artifacts into population metrics, time-based trends, and the EBS score. It reads like documentation, but it can also point at real metrics files from a run.

Weights & Biases is an optional observer for live telemetry, not the canonical source of truth for run scoring.

Artifact Source

Start with bundled sample artifacts, or point the page at a real metrics directory from data/runs/<run_id>/metrics.

Data Flow

From Events To Metrics

Every run writes an event stream to data/runs/<run_id>/events.jsonl. The builders then read those persisted artifacts after the run completes and derive summary metrics, per-tick metrics, and EBS component scores.

01 events.jsonl

Canonical record of decisions, outcomes, and agent state.

02 summary.json

Whole-run counts and rates for survival, actions, and innovation.

03 timeseries.jsonl

Tick-by-tick trends for the population over time.

04 ebs.json

Weighted interpretation layer built on top of those artifacts.

Run Summary

Population Metrics

These metrics answer what happened across the whole population by the end of the run: who survived, how cleanly actions resolved, and whether innovation attempts became usable behavior.

Survival rate = final survivors / initial population

Oracle success rate = successful oracle resolutions / total oracle resolutions

Innovation approval rate = approved innovations / innovation attempts

Innovation realization rate = approved innovations later used / approved innovations

Tick Metrics

Whole Run Vs Per-Tick Reading

Whole-run metrics tell you how a run ended. Per-tick metrics tell you how the population changed over time, including alive count, mean hunger, mean energy, actions, deaths, and innovation activity.

Interpretation Layer

EBS Breakdown

EBS is a weighted interpretation layer built on top of the recorded artifacts, not a magic intelligence score. 

EBS =
0.30 * Novelty
+ 0.20 * Utility
+ 0.20 * Realization
+ 0.15 * Stability
+ 0.15 * Autonomy

Trust Notes

Limits And Interpretation

Direct counts are more concrete than heuristic scores. EBS is a structured interpretation layer based on implemented signals, not an absolute truth.

Some sub-signals are intentionally partial. For example, self_generated_subgoals is currently fixed at 0.0.

Weights & Biases remains an optional observer rather than the official record for metrics or scoring.
