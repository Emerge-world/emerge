I've been thinking about this for longer than I can remember. What would it look like if you dropped language models into a world with real constraints — hunger, energy, survival — and let them figure it out? Not with hand-coded behaviors, not with reward functions, but with prompts and memory and the ability to invent new things. What would emerge?

This project is the attempt to find out.

## What I built

The first commit contained everything: a 50×50 grid world with water, land, and tree tiles. Agents with life, hunger, and energy stats that actually matter — run out of energy and you start losing life; get too hungry and the same thing happens. A tick loop that asks each agent what they want to do, hands it to an oracle for validation, and applies the effects. A base action set: move, eat, rest, innovate.

And crucially: the oracle. Not a rules engine, not a hardcoded validator, but an LLM that reasons about whether an action makes sense given the physics of the world. The oracle is also the thing that judges innovations — when an agent invents a new action that's never been tried before, the oracle decides if it's plausible. Results are cached as precedents, so the same situation always produces the same outcome.

Alongside the code: a complete `project-bible/` — twelve context documents covering every aspect of the system's design, from the architecture through the agent memory model, the oracle's validation logic, the LLM integration strategy, and the long-horizon phases I haven't built yet. Social interaction. Evolution. Visualization. A 5-phase roadmap for what this thing might eventually become.

The whole thing in a single first commit, as if the idea had been held in my head long enough that it came out nearly complete.

## Why it matters

The premise sounds simple: agents try to survive. But the simplicity is the point. I want emergence — behavior that wasn't programmed but that arises from the interaction of simple rules and intelligent reasoning. If I hardcode too much, I've built a game, not a world. If I leave too much unconstrained, nothing interesting happens because nothing is at stake.

The oracle is the key design decision. It's what lets the world be consistent without being predetermined. An oracle that reasons means that agents can invent things I haven't anticipated, and the world will handle them sensibly. That's the crack through which emergence enters.

The documentation — the project-bible — isn't just notes. It's the architecture of intent. Every design decision is recorded because design decisions compound. In six months, when I'm deep in Phase 3 and wondering why something works the way it does, I want to know what I was thinking in Phase 0.

## Things to consider

The whole thing rests on a 3-billion-parameter model running locally. That's the most constrained part of the system. The oracle, the agent decisions, the memory compression — all of it is bounded by what a small LLM can do reliably. Somewhere in the design there's a tension between what I want agents to be capable of and what the model can actually produce consistently.

I also started with a prompt-first approach to agent behavior, which means the agents are only as good as their prompts. The prompts are embedded in Python source code at this point, which is a technical debt I'm already aware of. Iteration on agent behavior means editing Python, which slows everything down. That's going to need to change.

And Phase 0 is stable, but "stable" means "runs without crashing," not "does interesting things." The agents move around, eat when they can, rest when they need to. They survive, mostly. They don't yet feel like they're learning, or planning, or building anything cumulative. That's Phase 1.

## What's next

Phase 1: Intelligence. The goal is to make the agents feel like they're actually thinking — not just reacting to each tick independently, but carrying knowledge forward, personalizing their behavior, improving their prompts, and eventually innovating in structured ways.

Memory first. Then better prompts. Then testing infrastructure, so changes can be evaluated and not just eyeballed. Then innovation guardrails, so when agents invent new actions, those actions are meaningful rather than arbitrary.

The world exists. Now it needs to become interesting.
